# 🐛 Bug Report: 订单生成时旅客信息错误，始终显示为当前登录用户

## 🐛 Bug描述
- 用户在预订车票时，若勾选了非本人的其他乘车人（如“ABC”），提交订单后，订单详情中的“旅客信息”会被错误地记录为当前登录用户的姓名（如“ABCD”）。
- 这意味着无论用户选择哪个乘车人，系统似乎都默认使用当前账户所有者的信息生成车票，导致实际乘车人信息丢失或错误，无法为他人购票。

## 📍 复现步骤
1. 登录系统。
2. 查询车票并点击“预订”，进入订单确认页面（页面：`frontend/src/views/ticket/LeftTicketRound.vue` 或相关组件）。
3. 在“乘车人”列表中，勾选一个**非当前登录用户本人**的乘车人（例如“ABC”）。
4. 提交订单。
5. 进入订单中心或支付页面查看该笔订单。
6. 观察“旅客信息”列，显示的姓名是当前登录用户（如“ABCD”），而非步骤3中选择的“ABC”。

## 🎯 期望行为
- 订单生成后，订单详情中的“旅客信息”（姓名、证件类型、证件号）必须与用户在预订界面勾选的乘车人完全保持一致。
- 系统应正确处理前端提交的乘车人列表，支持为列表中的任意乘客购票。

- 后端接口：`backend/app/api/v1/endpoints/orders.py` (接收订单请求)

## 📝 备注
- **重点排查方向**：
  1. 前端是否正确发送了 `passengers` 数组（包含“ABC”的 ID）。
  2. 后端在创建订单条目时，是否错误地使用了 `current_user` (当前登录用户) 的信息填充了乘客字段，而忽略了请求体中的乘客信息。
